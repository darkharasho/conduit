# Conduit v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Conduit v1 — a low-latency Linux input remapping daemon (evdev→engine→uinput) with per-app profiles (X11 + Hyprland) and a Tauri UI, per the approved spec at `docs/superpowers/specs/2026-07-11-conduit-design.md`.

**Architecture:** Cargo workspace with three crates + a Tauri app. `conduit-core` is a pure state machine (no I/O) implementing remaps, tap-hold (permissive hold), layers, and profile switching. `conduit-daemon` wires it to evdev/uinput/udev, a focus watcher, and a Unix-socket IPC server. `conduit-proto` holds shared IPC types. `ui/` is a Tauri 2 + React app that is a stateless view over the daemon.

**Tech Stack:** Rust (stable, edition 2021), crates: `evdev` (incl. uinput virtual devices), `udev`, `x11rb`, `serde`/`serde_json`/`toml`, `indexmap`, `regex`, `crossbeam-channel`, `thiserror`, `anyhow`. UI: Tauri 2, React 18, TypeScript, Vite, vitest.

## Global Constraints

- Rust stable ≥ 1.75, edition 2021, workspace resolver = "2".
- `conduit-core` has NO I/O dependencies. Allowed deps: `serde`, `indexmap`, `regex`, `thiserror` only. All time is `u64` microseconds passed in by the caller — never read a clock inside core.
- All vitest runs MUST use `--maxWorkers=2` (machine constraint from user CLAUDE.md).
- Virtual output devices are named `Conduit Virtual Keyboard` / `Conduit Virtual Mouse`. The device manager MUST never grab a device whose name starts with `Conduit Virtual` (feedback-loop protection).
- Config file path: `~/.config/conduit/conduit.toml`. IPC socket: `$XDG_RUNTIME_DIR/conduit.sock`, mode 0600.
- Default tap-hold timeout: 200 ms. Default panic chord: `["leftctrl", "leftalt", "backspace"]`. The panic chord is detected on PHYSICAL (pre-mapping) events and toggles suspend.
- Bad config never partially applies: parse → validate → compile → atomic swap.
- Commit after every task with a conventional-commit message. Tests must pass before each commit.
- Key events on the hot path must not allocate per-event in steady state (Vec buffers are reused; lookup tables are flat arrays).

```rust
pub fn from_name(name: &str) -> Option<Key> {
    let name = name.to_ascii_lowercase();
    let canonical = ALIASES.iter().find(|(a, _)| *a == name).map(|(_, c)| *c).unwrap_or(&name);
```

**Interfaces:**
- Consumes: `Key`, `keys::from_name` (Task 1).
- Produces:
  - `Action` enum: `Key(Key)`, `TapHold { tap: Key, hold: HoldAction, timeout_us: u64 }`, `LayerWhileHeld(u8)`, `LayerToggle(u8)`, `Disabled`, `Passthrough`; `HoldAction { Key(Key), Layer(u8) }`
  - `CompiledConfig { settings: Settings, profiles: Vec<CompiledProfile>, default_idx: usize }`
  - `Settings { tap_hold_timeout_us: u64, panic_chord: Vec<Key>, grab_all_keyboards: bool, grab_keyboards: Vec<String>, grab_mice: Vec<String> }`
  - `CompiledProfile { name: String, matcher: Option<Matcher>, layers: Vec<LayerMap>, layer_names: Vec<String> }` where `LayerMap = Box<[Option<Action>; 768]>` (index = evdev code; layer 0 is base)
  - `Matcher { process: Option<String>, class: Option<String>, title: Option<regex::Regex> }` with `fn matches(&self, f: &FocusFields) -> bool`; `FocusFields<'a> { process: &'a str, class: &'a str, title: &'a str }`
  - `pub fn compile(toml_str: &str) -> Result<CompiledConfig, ConfigError>` — the single entry point. `ConfigError` is a `thiserror` enum with variants: `Toml(String)`, `UnknownKey { profile: String, name: String }`, `UnknownLayer { profile: String, name: String }`, `UnknownInherit { profile: String, from: String }`, `InheritCycle(String)`, `PanicChordUnreachable { profile: String }`, `Empty` (no default profile and no keys).

Parsing rules (mirror the spec's TOML flavor exactly):
- Top level: `[settings]`, `[devices]`, `[profile.<name>]` (IndexMap — file order preserved; match precedence = file order, `default` always matched last as fallback).
- A key's action value is either a string — `"esc"` (emit key), `"layer:nav"` (invalid at top level of `keys` unless wrapped in tap_hold? No: a bare `"layer:nav"` string means `LayerToggle`), `"disabled"`, `"passthrough"` — or a table `{ tap = "...", hold = "...", timeout_ms = 200 }` meaning `TapHold`. In `hold`, `"layer:nav"` means `HoldAction::Layer`, any key name means `HoldAction::Key`. `timeout_ms` is optional, defaults to `settings.tap_hold_timeout`.
- `layers` tables define named layers; layer index assignment: base = 0, then named layers in file order starting at 1. `inherit = "name"` copies the parent's compiled layers first, then overlays this profile's `keys`/`layers` (same-named layer entries override per-key; new layers append).
- Validation: every key name must resolve via `keys::from_name`; every `layer:X` must reference a defined layer in that profile (after inheritance); inherit target must exist and not cycle; if a profile maps EVERY key of `settings.panic_chord` to something other than `Passthrough`, reject with `PanicChordUnreachable`. If no `[profile.default]` exists, synthesize an empty one (all passthrough).

```rust
use indexmap::IndexMap;
use serde::Deserialize;
use crate::{event::Key, keys};

pub const KEY_TABLE_SIZE: usize = 768; // > BTN_TASK (0x117); covers all key/button codes we handle

#[derive(Clone, Copy, PartialEq, Debug)]
pub enum HoldAction { Key(Key), Layer(u8) }

#[derive(Clone, Copy, PartialEq, Debug)]
pub enum Action {
    Key(Key),
    TapHold { tap: Key, hold: HoldAction, timeout_us: u64 },
    LayerWhileHeld(u8),
    LayerToggle(u8),
    Disabled,
    Passthrough,
}

pub type LayerMap = Box<[Option<Action>; KEY_TABLE_SIZE]>;

pub struct FocusFields<'a> { pub process: &'a str, pub class: &'a str, pub title: &'a str }

pub struct Matcher {
    pub process: Option<String>,
    pub class: Option<String>,
    pub title: Option<regex::Regex>,
}

impl Matcher {
    pub fn matches(&self, f: &FocusFields) -> bool {
        // All present fields must match (AND); comparison is case-insensitive
        // substring-free exact match for process/class, regex for title.
        let ok = |pat: &Option<String>, val: &str| pat.as_ref()
            .map_or(true, |p| p.eq_ignore_ascii_case(val));
        ok(&self.process, f.process)
            && ok(&self.class, f.class)
            && self.title.as_ref().map_or(true, |re| re.is_match(f.title))
    }
}

pub struct CompiledProfile {
    pub name: String,
    pub matcher: Option<Matcher>,
    pub layers: Vec<LayerMap>,      // [0] = base
    pub layer_names: Vec<String>,   // [0] = "base"
}

pub struct Settings {
    pub tap_hold_timeout_us: u64,
    pub panic_chord: Vec<Key>,
    pub grab_all_keyboards: bool,
    pub grab_keyboards: Vec<String>,
    pub grab_mice: Vec<String>,
}

pub struct CompiledConfig {
    pub settings: Settings,
    pub profiles: Vec<CompiledProfile>, // match precedence = this order, default last
    pub default_idx: usize,
}
```

Raw serde layer (private): `RawConfig { settings: RawSettings, devices: RawDevices, profile: IndexMap<String, RawProfile> }`; `RawProfile { r#match: Option<RawMatch>, inherit: Option<String>, keys: IndexMap<String, RawAction>, layers: IndexMap<String, IndexMap<String, RawAction>> }`; `RawAction` is `#[serde(untagged)] enum { Str(String), TapHold { tap: String, hold: String, timeout_ms: Option<u64> } }`.

Compile algorithm:
1. `toml::from_str::<RawConfig>` → map error to `ConfigError::Toml(msg)`.
2. Resolve settings (defaults: timeout 200 → store as µs; panic chord names → `Vec<Key>`; chord must be non-empty and all names valid).
3. Synthesize `default` profile if absent. Compile profiles in file order but resolve `inherit` first via depth-first resolution with a visiting set (`InheritCycle` on revisit; `UnknownInherit` if missing). Inheritance = deep-clone parent's compiled layers + layer_names, then overlay: for each named layer in the child that exists in parent, overwrite per-key; new named layers append (index = next free); child `keys` overlay base layer 0.
4. Action string parsing: `"disabled"` → Disabled; `"passthrough"` → Passthrough; `"layer:X"` → LayerToggle(idx of X, `UnknownLayer` if undefined); anything else → `keys::from_name` (`UnknownKey` if None) → `Action::Key`. TapHold table: tap must be a key name; hold is `"layer:X"` → `HoldAction::Layer` or key name → `HoldAction::Key`; timeout = `timeout_ms.unwrap_or(settings)` × 1000. In per-key layer maps inside `layers.<name>`, the same string grammar applies.
5. Ordering: profiles vector = non-default profiles in file order, then default appended last; `default_idx` = last index. (Engine walks the vec and takes the first match; default has `matcher: None` which matches everything.)
6. Panic-chord validation per profile: chord is unreachable only if EVERY chord key has a mapping in layer 0 that is not `Passthrough`. (Detection is pre-mapping at runtime; this validation is the spec's belt-and-suspenders rule.)

**Files:**

**Interfaces:**
- Consumes: `CompiledConfig`, `Action`, `Event`, `Key`, `KeyState` (Tasks 1–2).
- Produces:
  - `Engine::new(cfg: CompiledConfig) -> Engine`
  - `Engine::handle(&mut self, ev: Event) -> &[Event]` — process one physical event, return output events (slice into a reused internal buffer; cleared each call).
  - `Engine::tick(&mut self, now_us: u64) -> &[Event]` — resolve time-based state (used by Task 4).
  - `Engine::next_deadline_us(&self) -> Option<u64>` — when `tick` next needs calling (None = never).
  - Internal (established now, used by later tasks): `held: std::collections::HashMap<Key, HeldEntry>` where `HeldEntry { OutKey(Key), LayerHeld(u8), Swallowed }` — records what a physical press did so its release pairs with the same effect regardless of later profile/layer changes.

Semantics for this task: look up the pressed key in the active profile's base layer (layers come in Task 5 — for now only layer 0). `Action::Key(x)` → emit x press/release; `Passthrough`/unmapped → emit the event unchanged; `Disabled` → swallow (record `Swallowed` so the release is also swallowed); `Repeat` events follow whatever the held entry says. A release with no held entry passes through unchanged (key was down before we started).

```rust
use std::collections::HashMap;
use crate::config::{Action, CompiledConfig};
use crate::event::{Event, Key, KeyState};

#[derive(Clone, Copy, Debug)]
enum HeldEntry { OutKey(Key), LayerHeld(u8), Swallowed }

pub struct Engine {
    cfg: CompiledConfig,
    profile_idx: usize,
    active_layers: Vec<u8>,     // stack; index 0 is always base layer 0 (Task 5)
    held: HashMap<Key, HeldEntry>,
    out: Vec<Event>,            // reused output buffer
}

impl Engine {
    pub fn new(cfg: CompiledConfig) -> Self {
        let profile_idx = cfg.default_idx;
        Engine { cfg, profile_idx, active_layers: vec![0], held: HashMap::new(), out: Vec::with_capacity(16) }
    }

    pub fn handle(&mut self, ev: Event) -> &[Event] {
        self.out.clear();
        self.process(ev);
        &self.out
    }

    pub fn tick(&mut self, _now_us: u64) -> &[Event] { self.out.clear(); &self.out } // Task 4
    pub fn next_deadline_us(&self) -> Option<u64> { None }                            // Task 4

    fn lookup(&self, key: Key) -> Action {
        let profile = &self.cfg.profiles[self.profile_idx];
        for &layer in self.active_layers.iter().rev() {
            if let Some(a) = profile.layers[layer as usize][key.0 as usize] { return a; }
        }
        Action::Passthrough
    }

    fn process(&mut self, ev: Event) {
        match ev.state {
            KeyState::Press => match self.lookup(ev.key) {
                Action::Key(out) => {
                    self.held.insert(ev.key, HeldEntry::OutKey(out));
                    self.out.push(Event { key: out, ..ev });
                }
                Action::Passthrough => {
                    self.held.insert(ev.key, HeldEntry::OutKey(ev.key));
                    self.out.push(ev);
                }
                Action::Disabled => { self.held.insert(ev.key, HeldEntry::Swallowed); }
                _ => { /* TapHold: Task 4. Layer*: Task 5. */ }
            },
            KeyState::Release => match self.held.remove(&ev.key) {
                Some(HeldEntry::OutKey(out)) => self.out.push(Event { key: out, ..ev }),
                Some(HeldEntry::Swallowed) => {}
                Some(HeldEntry::LayerHeld(_)) => {} // Task 5
                None => self.out.push(ev),
            },
            KeyState::Repeat => match self.held.get(&ev.key) {
                Some(HeldEntry::OutKey(out)) => { let out = *out; self.out.push(Event { key: out, ..ev }); }
                Some(_) => {}
                None => self.out.push(ev),
            },
        }
    }

    #[cfg(test)]
    pub fn held_is_empty(&self) -> bool { self.held.is_empty() }
}
```

### Task 4: Tap-hold state machine (permissive hold, buffering, replay)

**Files:**
- Modify: `crates/conduit-core/src/engine.rs`

**Interfaces:**
- Consumes: everything from Task 3.
- Produces: full tap-hold behavior behind the same `handle`/`tick`/`next_deadline_us` API. Adds internal `pending: Option<Pending>` and `buffer: Vec<Event>`; adds `HeldEntry::HoldKey(Key)` and `HeldEntry::TapPending` semantics.

Exact semantics (from the spec — implement precisely these):
1. Press of a `TapHold` key → nothing emits; `pending = Some(Pending { phys: key, tap, hold, deadline: ev.time_us + timeout_us })`.
2. While pending, ALL subsequent input events go to `buffer` (not processed), EXCEPT the pending key's own release.
3. Resolve as TAP if the pending key's release arrives before the deadline: emit tap press at the original press time, tap release at the release time.
4. Resolve as HOLD if (a) `tick(now)` is called with `now >= deadline`, or (b) the buffer contains a press AND a release of the same key (permissive hold) — check after each buffered event arrives.
5. On resolution: apply the resolved action (tap → emit press[+release]; hold key → emit hold press + record `held[phys] = OutKey(hold_key)`; hold layer → push layer + record `LayerHeld`), then drain `buffer` through normal processing in order (which may establish a NEW pending — remaining buffered events re-buffer behind it). Buffered events keep their original `time_us`.
6. `next_deadline_us()` returns the pending deadline if any.
7. Repeat events for a pending key are swallowed.

```rust
// copy the pending key out first so we don't hold a borrow of self across mutations
if let Some(pending_phys) = self.pending.as_ref().map(|p| p.phys) {

        // pending key released before deadline → tap
        self.resolve(Resolution::Tap { release_time_us: ev.time_us });
```

`resolve()` — the one subtle function; drain via `std::mem::take(&mut self.buffer)` and feed each event back through `process()` (re-entrancy is safe because `pending` is `None` during the drain start; a buffered TapHold press re-establishes `pending` and subsequent drained events re-buffer via the block above):

```rust
enum Resolution { Tap { release_time_us: u64 }, Hold }

fn resolve(&mut self, how: Resolution) {
    let p = self.pending.take().expect("resolve without pending");
    match how {
        Resolution::Tap { release_time_us } => {
            self.out.push(Event { key: p.tap, state: KeyState::Press, time_us: p.press_time_us });
            self.out.push(Event { key: p.tap, state: KeyState::Release, time_us: release_time_us });
        }
        Resolution::Hold => match p.hold {
            crate::config::HoldAction::Key(k) => {
                self.held.insert(p.phys, HeldEntry::OutKey(k));
                self.out.push(Event { key: k, state: KeyState::Press, time_us: p.press_time_us });
            }
            crate::config::HoldAction::Layer(l) => {
                self.active_layers.push(l);
                self.held.insert(p.phys, HeldEntry::LayerHeld(l));
            }
        },
    }
    for ev in std::mem::take(&mut self.buffer) { self.process(ev); }
}
```

`tick(now)`: if `pending.deadline_us <= now` → `resolve(Hold)`. `next_deadline_us()`: `self.pending.as_ref().map(|p| p.deadline_us)`. Release of a `LayerHeld` entry pops that layer from `active_layers` (retain-remove last occurrence).

```rust
    let idx = self.cfg.profiles.iter().position(|p| p.matcher.as_ref().map_or(true, |m| m.matches(f)))
```

### Task 11: Run loop — grab, read, translate, emit

**Files:**
- Create: `crates/conduit-daemon/src/runloop.rs`

**Interfaces:**
- Consumes: `Engine` (core), `VirtualOutput` (Task 10), `devices` (Task 9).
- Produces the daemon's central message type and loop, which every later task feeds:

```rust
pub enum Msg {
    Input(conduit_core::event::Event),
    Focus(conduit_proto::FocusInfo),
    Reload(conduit_core::config::CompiledConfig),
    Suspend, Resume,
    DeviceAdded(PathBuf), DeviceRemoved(PathBuf),
    Subscribe(SubscribeKind, crossbeam_channel::Sender<conduit_proto::Push>),
    Query(QueryKind, crossbeam_channel::Sender<conduit_proto::Response>),
    CaptureNextKey(crossbeam_channel::Sender<conduit_proto::Response>),
}
```

Structure:
- `devices::spawn_reader(path, tx: Sender<Msg>) -> GrabHandle` — opens the evdev device, `grab()`s it, loops `fetch_events()`, converts each `EV_KEY` input event to core `Event` (timestamp: convert the kernel event's timeval to monotonic µs via a base offset captured at startup: `base_mono_us + (ev_time - base_wall_time)`; simpler and sufficient: use `Instant::now()` µs at read time), sends `Msg::Input`. Non-key events (EV_REL, EV_SYN, EV_MSC) from grabbed MICE must be forwarded straight to the virtual mouse — pass an `Arc<Mutex<VirtualOutput>>` clone to mouse readers for direct forwarding (motion latency must not pay a channel hop; the mutex is uncontended since the engine only writes buttons). Reader thread exits on read error (device unplugged) and sends `DeviceRemoved`.
- `runloop::run(engine: Engine, out: Arc<Mutex<VirtualOutput>>, rx: Receiver<Msg>)` — the engine thread:

```rust
loop {
    let timeout = engine.next_deadline_us().map(|d| Duration::from_micros(d.saturating_sub(now_us())));
    let msg = match timeout {
        Some(t) => match rx.recv_timeout(t) { Ok(m) => Some(m), Err(Timeout) => None, Err(Disconnected) => return },
        None => match rx.recv() { Ok(m) => Some(m), Err(_) => return },
    };
    let outputs: &[Event] = match msg {
        None => engine.tick(now_us()),
        Some(Msg::Input(ev)) => { forward_to_subscribers_pre(ev); engine.handle(ev) }
        Some(Msg::Focus(f)) => { engine.set_focus(&fields(&f)); push_status(); &[] }
        Some(Msg::Reload(cfg)) => engine.swap_config(cfg, &current_focus_fields()),
        Some(Msg::Suspend) => engine.suspend(),   // returns flush events
        // ... queries answered inline from engine state; subscribe pushes stored in Vec<Sender>
    };
    for ev in outputs { out.lock().unwrap().emit(ev)?; forward_to_subscribers_post(*ev); }
}
```

- Subscriber sends use `try_send`/bounded channels — a slow UI must NEVER block the hot path; on full/disconnected, drop the subscriber.
- `main.rs` order: load config → create `VirtualOutput` (BEFORE grabbing — spec requirement) → discover + grab + spawn readers → spawn runloop thread → (focus/IPC threads come in Tasks 12–14) → wait on runloop join.

```ts
export interface TesterRow {

  pre: { name: string; state: string; timeUs: number };
```
